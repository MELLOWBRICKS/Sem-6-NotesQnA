A. Explain Security Functionality-Ease of Use Triangle and CIA Triad. (5 marks)

The CIA Triad is fundamental in information security and consists of three principles:

  1. Confidentiality:
    • This principle ensures that information is only accessible to those who are authorized to view it.
    • Example: Encryption of sensitive files or using access controls.
    • Diagram: An image could show a locked file cabinet representing protected data.
  1. Integrity:
    • Integrity ensures that data remains accurate and unaltered unless by authorized users.
    • Example: Using checksums to verify data integrity during transmission.
    • Diagram: A shield icon could represent data integrity, highlighting checksums.
  1. Availability:
    • Availability ensures that systems and data are accessible when required.
    • Example: Backup systems and redundant server setups.
    • Diagram: A clock symbol could be used to indicate timely access to information.

B. Compare and Contrast CSS XSRF (5 marks)

  1. CSS (Cross-Site Scripting):
    • Attacks target end users by injecting scripts that execute within the user's browser.
    • Can steal data like cookies or tokens, as the script runs with the user's privileges.
  1. XSRF (Cross-Site Request Forgery):
    • Exploits the trust of a web application in the user, forcing it to perform unwanted actions on behalf of authenticated users.
    • Relies on authenticated sessions to execute unauthorized actions.

C. How does SQL Injection work? Explain any 2 payloads and preventions. (5 marks)

SQL injection allows unauthorized users to interfere with the queries that an application makes to its database.

Example Payloads:

  1. Authentication Bypass:
    username: admin' --
password: [anything]

    This input can trick the application into logging in the attacker without a valid password.

  1. Data Extraction:
    ' UNION SELECT username, password FROM users --

    This command combines results from the original query with sensitive user data.

Prevention Measures:

  1. Parameterized Queries:
    • Utilize prepared statements to ensure user inputs are treated strictly as data.
  1. Input Validation:
    • Rigorously validate and sanitize inputs to ensure they conform to expected formats.

D. Define the Term Vulnerability. (5 marks)

A vulnerability is defined as a weakness in a system that can be exploited by threats to gain unauthorized access or cause harm to systems. Vulnerabilities can be found in hardware, software, and organizational processes.

Types include:

  • Physical Vulnerabilities: Security weaknesses in facilities.
  • Software Vulnerabilities: Bugs or flaws in code.
  • Procedural Vulnerabilities: Weaknesses in organizational practices.

E. Explain XSS/CSS Attack and Brute Force Attack with 2 Preventions each. (5 marks)

XSS (Cross-Site Scripting):

XSS involves injecting malicious scripts into user browsers through vulnerable web applications, enabling attackers to steal cookies, session tokens, or sensitive information.

Prevention:

  1. Input Sanitization: Ensure all user inputs are sanitized to remove any potentially malicious code.
  1. Content Security Policy (CSP): Implement CSP to reduce the risk of XSS by controlling which resources can be loaded by the browser.

Brute Force Attack:

A brute force attack involves an attacker systematically guessing passwords until the correct one is found.

Prevention:

  1. Account Lockout Mechanisms: Implement procedures that lock user accounts after a certain number of failed login attempts.
  1. Strong Password Policies: Require the use of complex passwords to thwart automated guessing tools.

F. Distinguish between Denial of Service vs Distributed Denial of Service. (5 marks)

Denial of Service (DoS):

  • Originates from a single source.
  • Overloads the server with requests, making it unavailable to legitimate users.

Distributed Denial of Service (DDoS):

  • Originates from multiple compromised systems (botnets).
  • More complex and harder to mitigate due to multiple points of attack.

G. Elaborate DNS Poisoning. (5 marks)

DNS Poisoning:

This is a form of attack in which corrupt DNS data is inserted into the cache of a server. This leads users to malicious websites instead of the intended ones.

H. Write a note on Session Hijacking and explain the concept of session ID generation during an attack.

Session hijacking is an attack where an attacker takes control of a user's session by stealing or predicting a valid session token. This can lead to unauthorized actions being performed while masquerading as the legitimate user.

Session ID Generation:

  • Attackers can exploit weak session ID generation methods or steal session IDs through methods like XSS or packet sniffing.
  • Session IDs should be long, random, and regenerated after every action to mitigate risks.

I. Explain Buffer Overflow Attack. (5 marks)

A buffer overflow occurs when more data is written to a block of memory, or buffer, than it can hold, causing the excess data to overflow into adjacent memory. This could be exploited to execute arbitrary code.

How it Works:

  • An attacker can manipulate programs (especially those written in languages like C) to overflow buffers and alter the control flow to execute injected code.

Prevention:

  1. Use Safe Functions: Employ safe functions that limit input size.
  1. Heap Allocation and Bounds Checking: Use memory-safe programming practices.
