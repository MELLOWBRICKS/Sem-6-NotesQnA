Q1: Explain the role of controller deployment in OpenStack.

The Controller Node in OpenStack is the central management unit responsible for coordinating the entire cloud infrastructure. It provides authentication, networking, storage, and orchestration services.

Roles and Responsibilities of the Controller Node:

  1. Identity Management (Keystone)
    • Handles authentication and authorization for users and services.
  1. Dashboard (Horizon)
    • Provides a web-based interface for managing OpenStack components.
  1. Networking (Neutron)
    • Manages networking services, IP addresses, and virtual networks.
  1. Compute Scheduling (Nova-Scheduler)
    • Assigns virtual machines (VMs) to compute nodes based on resources.
  1. Storage Services (Cinder & Swift)
    • Cinder provides block storage, while Swift manages object storage.
  1. Message Queue (RabbitMQ)
    • Facilitates communication between OpenStack services.
  1. Image Service (Glance)
    • Stores and manages VM images for instance deployment.
  1. Orchestration (Heat)
    • Automates infrastructure deployment using templates.

Importance of Controller Node:

  • Acts as the brain of the OpenStack cloud.
  • Ensures efficient resource allocation and management.
  • Provides a centralized control panel for administrators.
  • Supports scalability and high availability in cloud environments.

Q2: Write a short note on Public Key Infrastructure (PKI) and Single Sign-On (SSO).

1. Public Key Infrastructure (PKI)

PKI is a framework for managing digital certificates and cryptographic keys to secure communications and transactions.

Components of PKI:

  • Certificate Authority (CA) – Issues and verifies digital certificates.
  • Registration Authority (RA) – Authenticates certificate requests.
  • Public & Private Keys – Used in encryption and authentication.
  • Digital Certificates – Verify the identity of users or devices.

Uses of PKI in Cloud Computing:

  • Secure data encryption using SSL/TLS protocols.
  • Digital signatures for authentication and integrity.
  • Identity verification for cloud users and services.

2. Single Sign-On (SSO)

SSO is an authentication mechanism that allows users to log in once and gain access to multiple applications or services without re-entering credentials.

Working of SSO:

  1. The user logs in once with a central authentication server.
  1. A token is issued and used to authenticate the user across multiple applications.
  1. The user gains seamless access to various services without repeated logins.

Examples of SSO:

  • Google SSO (Login once to access Gmail, Drive, YouTube).
  • Microsoft Azure SSO (Access Office 365, Teams, OneDrive).

Q3: What are the steps involved in deploying OpenStack in a production environment?

Deploying OpenStack for production requires careful planning and configuration.

Steps for OpenStack Production Deployment:

  1. Choose a Deployment Method
    • Manual installation (via CLI).
    • Automated deployment (using Ansible, Juju, or Packstack).
  1. Set Up Controller Node
    • Install OpenStack Identity (Keystone).
    • Configure Networking (Neutron).
    • Set up storage (Cinder & Swift).
  1. Deploy Compute Nodes
    • Install Nova Compute for virtual machine management.
    • Configure hypervisors (KVM, Xen, or VMware).
  1. Configure Storage Services
    • Use Cinder for block storage.
    • Use Swift for object storage.
  1. Set Up Security and Access Control
    • Implement Identity and Access Management (IAM).
    • Use firewalls and security groups for protection.
  1. Monitor and Optimize Performance
    • Use Telemetry (Ceilometer) to track usage and performance.
    • Optimize load balancing and scaling.

Q5: Write a short note on Hashing and Digital Signature.

1. Hashing

Hashing is a one-way cryptographic process that converts data into a fixed-length hash value.

Properties of Hashing:

  • Deterministic – The same input always produces the same output.
  • Irreversible – Cannot retrieve original data from hash.
  • Fast Computation – Efficient processing even for large data.

Examples of Hashing Algorithms:

  • MD5 (128-bit hash, weak security).
  • SHA-256 (256-bit hash, strong security).

Uses in Cloud Security:

  • Password storage (hashed passwords instead of plaintext).
  • Data integrity verification.

2. Digital Signature

A digital signature is a cryptographic technique that ensures:

  1. Authentication – Confirms the sender’s identity.
  1. Integrity – Ensures data is not altered.
  1. Non-repudiation – Prevents the sender from denying they signed a document.

Working of Digital Signature:

  1. Sender hashes the message.
  1. The hash is encrypted using the sender’s private key.
  1. The receiver verifies the signature using the sender’s public key.

Example:

  • RSA Digital Signature Algorithm (DSA) used in secure email communication.

Q6: What is SSO? What are the advantages of SSO?

Single Sign-On (SSO) is an authentication process that allows users to log in once and access multiple applications without repeated credential entry.

Advantages of SSO:

  1. User Convenience – No need to remember multiple passwords.
  1. Enhanced Security – Reduces the risk of weak passwords.
  1. Improved Productivity – Saves time by eliminating multiple logins.
  1. Centralized Control – Admins can manage user access more efficiently.
  1. Reduces IT Costs – Fewer password reset requests.

Examples: Google SSO, Facebook Login, Microsoft Azure SSO.

Q7: What is Identity and Access Management (IAM)?

Identity and Access Management (IAM) is a security framework that controls who can access cloud resources and what actions they can perform.

Components of IAM:

  1. Authentication – Verifies user identity (via passwords, biometrics, MFA).
  1. Authorization – Grants permissions to users based on roles.
  1. User Management – Admins can add, modify, or remove user accounts.
  1. Access Control Policies – Defines rules for accessing cloud resources.

Benefits of IAM in Cloud Computing:

  • Improves security by restricting unauthorized access.
  • Ensures compliance with regulations (e.g., GDPR, HIPAA).
  • Enhances operational efficiency through centralized access control.

Examples: AWS IAM, Azure Active Directory, Google Cloud IAM.

Q8: What are Cloud-Based Security Groups?

Security Groups in cloud computing are firewall-like rules that control inbound and outbound network traffic for virtual machines (VMs).

Features of Security Groups:

  1. Traffic Filtering – Define allow/deny rules for IP addresses and ports.
  1. Stateful Firewall – If incoming traffic is allowed, the response traffic is automatically permitted.
  1. Granular Access Control – Allows restrictions at the instance level.
  1. Dynamic Updates – Security group rules can be modified in real-time.

Example in OpenStack:

  • Create a security group rule allowing SSH access (port 22):
    openstack security group rule create --protocol tcp --dst-port 22 --remote-ip 0.0.0.0/0 SECURITY_GROUP_ID

Benefits:

  • Enhances network security for cloud applications.
  • Prevents unauthorized access to cloud instances.
