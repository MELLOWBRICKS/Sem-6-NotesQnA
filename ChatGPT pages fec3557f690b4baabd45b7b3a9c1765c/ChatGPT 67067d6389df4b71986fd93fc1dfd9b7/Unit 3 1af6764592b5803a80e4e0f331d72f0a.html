Q1. Describe the methods that are used by AlarmManager Class while creating any alarm application in Android.

What is AlarmManager?

AlarmManager is a system service that schedules tasks to run at a specific time or repeat at intervals even when the app is closed. It is commonly used for reminders, notifications, and background tasks.

Methods of AlarmManager:

MethodDescription
setExact()Triggers an alarm at an exact time.
setRepeating()Triggers an alarm at regular intervals.
setAndAllowWhileIdle()Allows alarms to run even in Doze mode.
setAlarmClock()Used for alarm clock applications.
cancel()Cancels a previously set alarm.

Example: Setting an Alarm

AlarmManager alarmManager = (AlarmManager) getSystemService(ALARM_SERVICE);
PendingIntent pendingIntent = PendingIntent.getBroadcast(this, 0, intent, 0);
alarmManager.setExact(AlarmManager.RTC_WAKEUP, triggerTime, pendingIntent);

🔹 Key Points:

  • RTC_WAKEUP wakes the device at the scheduled time.
  • PendingIntent defines what happens when the alarm triggers.

Q5. What is AlarmManager? Explain what is Elapsed Real Time, Real Time Clock. Define with syntax the methods of AlarmManager class.

What is AlarmManager?

AlarmManager schedules tasks even when the app is not running. It uses two time systems:

  1. Elapsed Real Time (ELAPSED_REALTIME, ELAPSED_REALTIME_WAKEUP)
    • Time since device boot (ignores changes in system time).
    • Use case: Scheduling periodic background sync.
  1. Real Time Clock (RTC, RTC_WAKEUP)
    • Based on the actual clock time (affected by user changes).
    • Use case: Scheduling alarms, reminders.

Common AlarmManager Methods

alarmManager.set(AlarmManager.RTC_WAKEUP, triggerTime, pendingIntent);
alarmManager.setRepeating(AlarmManager.ELAPSED_REALTIME_WAKEUP, interval, pendingIntent);

🔹 RTC_WAKEUP wakes the device, while ELAPSED_REALTIME_WAKEUP is based on uptime.

Q2. Create an Android application to play video files in Android

What is Video Playback in Android?

Android provides the VideoView class for playing videos from local storage or URLs. It also supports MediaController for play, pause, and seek controls.

Main Code Snippet

videoView.setVideoURI(Uri.parse("android.resource://" + getPackageName() + "/" + R.raw.sample_video));
videoView.setMediaController(new MediaController(this));
videoView.start();

🔹 Key Points:

  • setVideoURI() loads the video.
  • setMediaController() adds playback controls.
  • start() begins playback.

Q4. Write a program to show use of MediaController.

What is MediaController?

MediaController provides built-in media playback controls (play, pause, seek) for VideoView.

Main Code Snippet

MediaController mediaController = new MediaController(this);
mediaController.setAnchorView(videoView);
videoView.setMediaController(mediaController);

🔹 Key Points:

  • setAnchorView() binds the controller to the video.
  • setMediaController() enables playback controls.

Q9. Write a Program to Play Video in Android and Also State the Use of setAnchorView.

What is setAnchorView()?

  • setAnchorView() positions the MediaController relative to the VideoView.
  • Ensures that controls are displayed on top of the video.

Main Code Snippet

videoView.setVideoURI(Uri.parse("android.resource://" + getPackageName() + "/" + R.raw.sample_video));

MediaController mediaController = new MediaController(this);
mediaController.setAnchorView(videoView);

videoView.setMediaController(mediaController);
videoView.start();

🔹 Key Points:

  • MediaController is anchored to VideoView for proper alignment.
  • The user can pause, play, and seek the video.

Q3. Explain how Shared Preferences are different from Saved Instance State?

Shared Preferences vs. Saved Instance State

FeatureShared PreferencesSaved Instance State
UsageStores persistent dataSaves temporary UI data
ScopeAvailable even after app restartsLost when app is killed
StorageStored as XML filesStored in Bundle
Use CaseSaving user settings, login infoRetaining text in a form after screen rotation

Example:

  • Shared Preferences: Storing login state
    SharedPreferences prefs = getSharedPreferences("MyPrefs", MODE_PRIVATE);
prefs.edit().putBoolean("isLoggedIn", true).apply();
  • Saved Instance State: Restoring text in EditText
    @Override
protected void onSaveInstanceState(Bundle outState) {
    outState.putString("userInput", editText.getText().toString());
    super.onSaveInstanceState(outState);
}

Q6. Explain Different Modes of Storage in Android.

Android provides multiple storage options for saving data:

  1. Shared Preferences – Stores key-value pairs (small data like settings).
  1. Internal Storage – Private app files stored in device memory.
  1. External Storage – Stores large files like images, videos (user can delete them).
  1. SQLite Database – Stores structured data in tables.
  1. Cloud Storage – Uses Firebase or external APIs for storage.

Q7. Explain Internal Storage in Android and its Useful Methods.

What is Internal Storage?

Internal Storage saves private app files that cannot be accessed by other apps.

Common Methods

MethodDescription
openFileOutput()Writes data to a file.
openFileInput()Reads data from a file.
deleteFile()Deletes a file.

Example: Writing to Internal Storage

FileOutputStream fos = openFileOutput("example.txt", MODE_PRIVATE);
fos.write("Hello World".getBytes());
fos.close();

Q8. Explain Below Given Modes of Shared Preferences with Syntax.

  1. MODE_APPEND – Appends new data to existing preferences.
    getSharedPreferences("MyPrefs", MODE_APPEND);
  1. MODE_ENABLE_WRITE_AHEAD_LOGGING – Enables faster data writes.
    getSharedPreferences("MyPrefs", MODE_ENABLE_WRITE_AHEAD_LOGGING);
  1. MODE_MULTI_PROCESS – Allows multiple processes to access shared preferences (deprecated).
    getSharedPreferences("MyPrefs", MODE_MULTI_PROCESS);
  1. MODE_PRIVATE – Restricts access to only the app itself (default mode).
    getSharedPreferences("MyPrefs", MODE_PRIVATE);
  1. MODE_WORLD_READABLE – Allows other apps to read preferences (deprecated).
  1. MODE_WORLD_WRITEABLE – Allows other apps to modify preferences (deprecated).
