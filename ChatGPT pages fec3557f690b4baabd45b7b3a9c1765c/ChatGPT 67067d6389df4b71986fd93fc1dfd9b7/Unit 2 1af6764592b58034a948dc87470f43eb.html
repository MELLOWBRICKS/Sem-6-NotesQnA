Q1. List and Explain the Elements for Menu in Android

A menu in Android is a collection of UI options that provide actions to the user. Android supports three types of menus:

1. Options Menu

  • The primary menu for an activity, shown when the user presses the menu button.
  • Declared in res/menu/ directory as an XML file.

2. Context Menu

  • A floating menu that appears when a long press is performed on a view.
  • Used for actions related to a specific UI element.

3. Popup Menu

  • A small floating menu that anchors to a specific view.
  • Appears when a button is clicked and disappears automatically when an item is selected.

Common Menu Elements

ElementDescription
<menu>The root element of a menu file.
<item>Defines a menu item. Contains attributes like title, id, and icon.
<group>Groups multiple items together to apply properties like visibility.
<submenu>A nested menu within an item.

Q2. Explain Popup Menu in Android with Example

A Popup Menu is a temporary menu that appears when a user taps a button. It does not persist and disappears when the user selects an item.

Example: Implementing Popup Menu

PopupMenu popup = new PopupMenu(this, button);
popup.getMenuInflater().inflate(R.menu.popup_menu, popup.getMenu());

popup.setOnMenuItemClickListener(item -> {
    if (item.getItemId() == R.id.option1) {
        Toast.makeText(this, "Option 1 selected", Toast.LENGTH_SHORT).show();
    }
    return true;
});

popup.show();

🔹 Key Points:

  • PopupMenu is attached to a view (button).
  • setOnMenuItemClickListener() handles item clicks.

Q3. Explain the Following Events in Intent Class in Android

1. android.intent.action.BATTERY_CHANGED

  • Broadcast when the battery level or charging status changes.
  • Apps can register for this event to monitor battery percentage.

2. android.intent.action.BATTERY_LOW

  • Sent when the battery falls below a critical level.
  • Apps should reduce power usage (e.g., stop background tasks).

3. android.intent.action.BATTERY_OKAY

  • Sent when the battery recovers from a low state.
  • Apps can resume normal operations.

4. android.intent.action.BOOT_COMPLETED

  • Broadcast after the device finishes booting.
  • Used by apps that need to start automatically after reboot.

5. android.intent.action.BUG_REPORT

  • Used to send bug reports to developers.
  • Triggered when a user reports an issue.

6. android.intent.action.CALL

  • Used to initiate a phone call from an app.
  • Requires CALL_PHONE permission.
Intent callIntent = new Intent(Intent.ACTION_CALL);
callIntent.setData(Uri.parse("tel:+911234567890"));
startActivity(callIntent);

A ContextMenu is a floating menu that appears on long press on a UI element.

registerForContextMenu(myButton);

Q5. Explain Screen Navigation in Android

Screen navigation in Android refers to moving between different screens (activities/fragments) within an app. This can be done using buttons, menus, gestures, or the system back button.

Types of Navigation:

  1. Up Navigation (Parent-Child Relationship)
    • Moves to the parent activity.
    • Example:
      getSupportActionBar().setDisplayHomeAsUpEnabled(true);
  1. Back Navigation (History Stack)
    • Moves to the previous screen using the back button.
    • Example:
      java
CopyEdit
@Override
public void onBackPressed() {
    super.onBackPressed();
}
  1. Intent-Based Navigation (Explicit Intent)
    • Moves between activities.
    • Example:
      java
CopyEdit
Intent intent = new Intent(MainActivity.this, SecondActivity.class);
startActivity(intent);
  1. Fragment Navigation (Single Activity Apps)
    • Switches fragments instead of activities.
    • Example:
      java
CopyEdit
getSupportFragmentManager().beginTransaction()
    .replace(R.id.fragment_container, new ExampleFragment())
    .commit();
  1. Bottom Navigation (Tab-Based UI)
    • Uses a bottom bar to switch fragments.
    • Example:
      java
CopyEdit
bottomNavigationView.setOnNavigationItemSelectedListener(item -> {
    loadFragment(new HomeFragment());
    return true;
});
  1. Navigation Drawer (Sliding Menu)
    • Uses a side panel for navigation.
    • Example:
      java
CopyEdit
drawerLayout.openDrawer(GravityCompat.START);

Conclusion:

Navigation helps users move within the app efficiently. Methods vary based on UI structure, including intents, fragments, bottom navigation, and navigation drawers.

Q6. Explain What is Notification in Android

A notification is a message that appears outside the app UI to inform the user about events (e.g., new message, update).

Example: Creating a Basic Notification

NotificationCompat.Builder builder = new NotificationCompat.Builder(this, "channel_id")
        .setSmallIcon(R.drawable.ic_notification)
        .setContentTitle("New Message")
        .setContentText("You have a new message")
        .setPriority(NotificationCompat.PRIORITY_DEFAULT);

NotificationManagerCompat manager = NotificationManagerCompat.from(this);
manager.notify(1, builder.build());

🔹 Key Points:

  • NotificationCompat.Builder is used to create notifications.
  • notify() displays the notification.

Q7. Explain AsyncTask and AsyncTaskLoader

1. AsyncTask

  • A helper class for performing background tasks.
  • Consists of three main methods:
    • doInBackground() – Runs in the background.
    • onPreExecute() – Runs before execution starts.
    • onPostExecute() – Runs after execution is finished.

Example:

class DownloadTask extends AsyncTask<String, Void, String> {
    protected String doInBackground(String... urls) {
        return "Downloading...";
    }
    protected void onPostExecute(String result) {
        textView.setText(result);
    }
}

2. AsyncTaskLoader

  • A more efficient alternative to AsyncTask.
  • Survives configuration changes (e.g., screen rotation).

Example:

@Override
public String loadInBackground() {
    return "Loading data...";
}

Q8. Define Option Menu in Android with a Program

The Options Menu is the primary menu in Android, appearing at the top-right corner.

Example: Creating an Options Menu

@Override
public boolean onCreateOptionsMenu(Menu menu) {
    getMenuInflater().inflate(R.menu.options_menu, menu);
    return true;
}
@Override
public boolean onOptionsItemSelected(MenuItem item) {
    if (item.getItemId() == R.id.settings) {
        Toast.makeText(this, "Settings clicked", Toast.LENGTH_SHORT).show();
        return true;
    }
    return super.onOptionsItemSelected(item);
}

🔹 Key Points:

  • onCreateOptionsMenu() inflates the menu.
  • onOptionsItemSelected() handles menu item clicks.

Q9. Explain Android Unbounded Service Lifecycle

A Service is a background process in Android. An Unbounded Service is one that runs independently of the calling component and does not return results.

Lifecycle of an Unbounded Service

  1. onCreate() – Called when the service is first created.
  1. onStartCommand() – Called every time a client starts the service.
  1. onDestroy() – Called when the service is stopped.

Example: Implementing an Unbounded Service

public class MyService extends Service {
    @Override
    public int onStartCommand(Intent intent, int flags, int startId) {
        return START_STICKY;
    }
    @Override
    public void onDestroy() {
        super.onDestroy();
    }
}

🔹 Key Points:

  • START_STICKY keeps the service running even if the app is closed.
  • The service runs independently without binding to any activity.
