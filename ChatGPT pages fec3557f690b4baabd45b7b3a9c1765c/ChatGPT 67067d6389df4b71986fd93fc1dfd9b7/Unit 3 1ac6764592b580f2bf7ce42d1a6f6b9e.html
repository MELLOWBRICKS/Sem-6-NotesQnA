Q1: What is OpenStack Command Line Interface (CLI)?

The OpenStack Command-Line Interface (CLI) is a tool that allows users to interact with OpenStack services through a command-line terminal. It helps manage cloud resources such as compute instances, storage, and networking components.

Features of OpenStack CLI:

  1. Automation & Scripting – Enables users to automate cloud operations using scripts.
  1. Remote Management – Can be accessed remotely via SSH.
  1. Multiple Service Control – Manages various OpenStack services like Nova, Cinder, Neutron, and Swift.
  1. Efficient Resource Allocation – Provides real-time control over instances, networks, and storage.
  1. Flexibility – Supports authentication using OpenStack Identity Service (Keystone).

Common OpenStack CLI Commands:

  • List all available commands:
    openstack help
  • View running instances:
    openstack server list
  • Create a new instance:
    openstack server create --image IMAGE_NAME --flavor FLAVOR_NAME --network NETWORK_NAME INSTANCE_NAME
  • Delete an instance:
    openstack server delete INSTANCE_NAME
  • Check quota usage:
    openstack quota show

Q3: Write a short note on Cinder and Swift in OpenStack.

Cinder and Swift are OpenStack storage components that provide different types of storage solutions.

1. Cinder (Block Storage Service)

  • Provides persistent block storage for virtual machines.
  • Works like traditional hard drives, allowing direct attachment to instances.
  • Supports snapshots, backups, and volume management.
  • Commonly used with databases, file systems, and applications that require low-latency storage.

Example CLI Commands:

  • Create a volume:
    openstack volume create --size 10 my_volume
  • Attach a volume to an instance:
    openstack server add volume INSTANCE_NAME VOLUME_NAME

2. Swift (Object Storage Service)

  • Stores unstructured data (e.g., backups, images, videos, logs).
  • Highly scalable and optimized for redundancy and high availability.
  • Provides an HTTP-based API for easy access.
  • Suitable for content delivery networks (CDN), backup, and archiving.

Example CLI Commands:

  • Upload a file to Swift storage:
    openstack object create CONTAINER_NAME file.txt
  • List all stored objects:
    openstack object list CONTAINER_NAME

Q4: Write down the steps to Install and Configure OpenStack.

OpenStack installation can be done using DevStack, which is a lightweight tool to deploy OpenStack quickly.

Steps to Install OpenStack Using DevStack:

  1. Update System Packages:
    sudo apt update -y && sudo apt upgrade -y
  1. Create a New User for OpenStack:
    sudo useradd -s /bin/bash -d /opt/stack -m stack
echo "stack ALL=(ALL) NOPASSWD: ALL" | sudo tee /etc/sudoers.d/stack
  1. Switch to Stack User:
    sudo su - stack
  1. Clone DevStack Repository:
    git clone https://opendev.org/openstack/devstack
cd devstack
  1. Create Configuration File (local.conf):Add the following content:
    nano local.conf

    [[local|localrc]]
ADMIN_PASSWORD=StrongAdminSecret
DATABASE_PASSWORD=$ADMIN_PASSWORD
RABBIT_PASSWORD=$ADMIN_PASSWORD
SERVICE_PASSWORD=$ADMIN_PASSWORD
HOST_IP=192.168.10.100
  1. Run Installation Script:
    ./stack.sh
  1. Access OpenStack Dashboard:
    • Open a browser and go to http://<HOST_IP>/dashboard.
    • Login with admin credentials.

Q5: Write a short note on OpenStack.

OpenStack is an open-source cloud computing platform designed for building and managing private and public clouds. It provides Infrastructure-as-a-Service (IaaS) by controlling large pools of compute, storage, and networking resources.

Key Components of OpenStack:

  1. Nova – Manages compute instances (VMs).
  1. Glance – Manages VM images.
  1. Neutron – Provides networking services.
  1. Cinder – Block storage management.
  1. Swift – Object storage system.
  1. Horizon – Web-based OpenStack dashboard.
  1. Keystone – Identity management and authentication.
  1. Heat – Orchestrates infrastructure deployment.

Advantages of OpenStack:

  • Scalable and Flexible cloud computing solution.
  • Supports multi-tenancy for resource sharing.
  • Compatible with various hardware and virtualization platforms.
  • Enables automation and orchestration of cloud infrastructure.

Q6: Explain the concept of Quota in OpenStack.

Quota in OpenStack defines the maximum amount of cloud resources a user or tenant can consume. It helps administrators control resource usage to prevent excessive consumption by any single user.

Types of Quotas in OpenStack:

  1. Compute Quotas (Nova Service)
    • Limits the number of instances, vCPUs, and RAM per project.
    • Example CLI Command:
      openstack quota show --compute
  1. Storage Quotas (Cinder & Swift)
    • Controls the maximum storage space available to a user.
    • Example CLI Command:
      openstack quota show --volume
  1. Network Quotas (Neutron Service)
    • Restricts the number of networks, routers, and floating IPs.
    • Example CLI Command:
      openstack quota show --network

Benefits of Using Quotas:

  • Prevents resource overuse by individual users.
  • Optimizes cloud performance by balancing workloads.
  • Enhances security and stability by avoiding resource exhaustion.
