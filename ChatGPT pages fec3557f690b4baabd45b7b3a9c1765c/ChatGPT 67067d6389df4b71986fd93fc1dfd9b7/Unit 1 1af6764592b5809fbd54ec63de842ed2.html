Q1. List the methods in the activity lifecycle with an appropriate diagram

The Android Activity Lifecycle defines how an activity transitions between different states during its lifetime. The system invokes lifecycle callback methods when an activity is created, started, resumed, paused, stopped, or destroyed.

Activity Lifecycle Methods

  1. onCreate()
    • Called when the activity is first created.
    • Used for initialization (e.g., setting up UI, loading data).
  1. onStart()
    • Called when the activity becomes visible to the user.
    • UI updates and animations can be started here.
  1. onResume()
    • Called when the activity is in the foreground and ready for interaction.
    • Ideal for starting UI updates, animations, and resuming paused resources (like a video).
  1. onPause()
    • Called when another activity comes in front of the current activity.
    • Save user input or pause animations and processes that consume CPU.
  1. onStop()
    • Called when the activity is no longer visible.
    • Used to release resources, stop services, or save data.
  1. onRestart()
    • Called when the activity is restarted after being stopped.
  1. onDestroy()
    • Called before the activity is completely removed from memory.
    • Used for final cleanup of resources.

Q2. Android App with Checkboxes for Food Items and Order Button

Implementation Details:

  • The app includes CheckBoxes for food items with prices.
  • A Button displays the selected items and total price using Toast.

Main Code Snippet:

public void onClick(View view) {
    int totalAmount = 0;
    StringBuilder orderSummary = new StringBuilder("Selected Items:\n");

    if (pizzaCheckBox.isChecked()) {
        orderSummary.append("Pizza - ₹100\n");
        totalAmount += 100;
    }
    if (burgerCheckBox.isChecked()) {
        orderSummary.append("Burger - ₹80\n");
        totalAmount += 80;
    }
    if (coffeeCheckBox.isChecked()) {
        orderSummary.append("Coffee - ₹50\n");
        totalAmount += 50;
    }

    orderSummary.append("Total Amount: ₹").append(totalAmount);
    Toast.makeText(getApplicationContext(), orderSummary.toString(), Toast.LENGTH_LONG).show();
}

Q3. Explain the following fragment methods

Fragment Lifecycle Methods

  1. onAttach()
    • Called when the fragment is attached to the activity.
    • Used for initialization that requires the parent activity’s context.
  1. onActivityCreated()
    • Called after onCreateView(), ensuring the activity is fully created.
    • Used for UI initialization and setting up adapters or data sources.
  1. onResume()
    • Called when the fragment is visible and interactive.
    • Used for animations, user inputs, and UI updates.
  1. onDestroyView()
    • Called when the fragment’s view is being destroyed.
    • Used to clean up UI components to avoid memory leaks.
  1. onCreate()
    • Called to initialize essential data before creating the UI.
    • Used for setting up variables, database connections, etc.
  1. onPause()
    • Called when the fragment is no longer in the foreground.
    • Used to save user progress or stop background tasks.

Q9. What is Android Intent? Write a program on Implicit Intent.

Definition:

An Intent in Android is a messaging object used to communicate between different app components. It is mainly used for activity navigation, starting services, sending broadcasts, and interacting with system components. Intents are the backbone of Android app communication.

Types of Intents

  1. Explicit Intent – Used when navigating between components within the same app.
  1. Implicit Intent – Used when the system decides which app should handle the request.

Implicit Intent Code Snippet:

Intent intent = new Intent(Intent.ACTION_VIEW, Uri.parse("https://www.google.com"));
startActivity(intent);

Explicit Intent (Known Destination)

Used to navigate within the app (e.g., opening another activity).

Example:

Intent intent = new Intent(MainActivity.this, SecondActivity.class);
startActivity(intent);

Differences Between Explicit and Implicit Intents

FeatureExplicit IntentImplicit Intent
UsageWithin the same appAcross different apps
TargetSpecific activity/service is mentionedSystem decides the app

Conclusion

  • Explicit Intents navigate within an app.
  • Implicit Intents let the system pick the right app for the action.
  • Intent Filters enable an app to respond to implicit intents.
  • Broadcast Intents allow communication across apps.
