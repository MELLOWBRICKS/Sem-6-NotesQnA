Q1: Discuss classification or taxonomy of virtualization at different levels.

Virtualization can be classified into different categories based on the level at which it operates.

1. Execution Virtualization

  • Creates an execution environment separate from the host system.
  • Enables running multiple operating systems or applications.
  • Examples: VMware, VirtualBox.

2. Hardware-Level Virtualization

  • Also called System Virtualization, it provides a virtualized hardware environment.
  • A hypervisor (VMM) manages the virtual machines (VMs).
  • Types:
    • Full Virtualization – Complete hardware emulation, runs unmodified OS.
    • Para-Virtualization – OS is modified to work with the hypervisor.
    • Partial Virtualization – Some hardware components are virtualized.

3. Operating System-Level Virtualization

  • No hypervisor; the OS kernel allows multiple isolated instances (containers).
  • Example: Docker, LXC (Linux Containers).

4. Programming Language-Level Virtualization

  • Creates a runtime environment for executing programs across multiple platforms.
  • Examples: Java Virtual Machine (JVM), .NET Framework.

5. Application-Level Virtualization

  • Virtualizes applications instead of the whole system.
  • Runs software in a separate environment, avoiding conflicts with other applications.
  • Example: Citrix XenApp.

6. Storage Virtualization

  • Abstracts multiple storage devices into a single storage unit.
  • Improves data management, backup, and recovery.
  • Example: SAN (Storage Area Network).

7. Network Virtualization

  • Logical division of a network into multiple virtual networks.
  • Increases network efficiency and security.
  • Example: Virtual LANs (VLANs), Software-Defined Networking (SDN).

Q2: Explain the distinct features of virtualized environments.

A virtualized environment provides a simulated version of computing resources such as hardware, storage, and networks.

1. Guest

  • System component that interacts with the virtualization layer (VMs, applications).

2. Host

  • The physical machine where virtualization occurs.

3. Virtualization Layer

  • Provides an interface between the guest and host, allowing resource sharing.

Key Features:

  1. Increased Security – Virtual machines (VMs) are isolated, preventing security breaches.
  1. Managed Execution – Ensures controlled execution of applications across different environments.
  1. Sharing – Multiple virtual machines run on a single host efficiently.
  1. Aggregation – Multiple physical resources are combined into a single virtual entity.
  1. Emulation – A VM can mimic different hardware or OS environments.
  1. Isolation – Each VM operates independently without affecting others.
  1. Portability – VMs can be moved across different systems without modification.

Q3: Write a short note on oVirt.

oVirt is an open-source virtualization management platform that provides centralized management for virtualized environments.

Key Components of oVirt:

  1. oVirt Node – Computes virtualization units running VMs.
  1. oVirt Engine – The control unit for managing virtualization infrastructure.

Features of oVirt:

  • Enterprise-Level Virtualization – Built on KVM (Kernel-Based Virtual Machine).
  • Open-Source Alternative to VMware vSphere.
  • Supports Multiple Hypervisors – Manages KVM-based environments efficiently.
  • Scalable and Flexible – Suitable for both small and large-scale deployments.
  • Integration with Cloud Environments – Works with OpenStack.

oVirt vs. OpenStack:

  • oVirt focuses on virtualization management, while OpenStack provides IaaS (Infrastructure as a Service).

Q4: Explain different types of virtualization.

Virtualization is classified into different types based on the level of abstraction:

  1. Hardware Virtualization
    • Uses Hypervisors to create and manage virtual machines.
    • Types: Full, Para, and Partial Virtualization.
  1. Operating System Virtualization
    • Multiple OS instances run on a single kernel.
    • Example: Docker, LXC.
  1. Network Virtualization
    • Creates multiple logical networks over a single physical infrastructure.
    • Example: VLAN, SDN.
  1. Storage Virtualization
    • Abstracts physical storage into logical storage units.
    • Example: Storage Area Network (SAN).
  1. Application Virtualization
    • Applications run in an isolated environment without installing them locally.
    • Example: Citrix XenApp, VMware.
  1. Desktop Virtualization
    • Provides virtual desktops instead of physical desktops.
    • Example: Virtual Desktop Infrastructure (VDI).

Q5: Explain the pros and cons of virtualization.

Advantages of Virtualization:

  1. Cost Savings – Reduces hardware and maintenance costs.
  1. Efficient Resource Utilization – Maximizes the use of computing resources.
  1. Scalability – Easily scales up or down as per demand.
  1. Security – Isolates virtual machines, reducing security risks.
  1. Disaster Recovery – Simplifies backup and recovery processes.

Disadvantages of Virtualization:

  1. Performance Overhead – Virtualized systems introduce latency due to resource sharing.
  1. Security Risks – If one VM is compromised, others can be affected.
  1. Complex Management – Requires specialized skills for setup and maintenance.
  1. Hardware Dependency – Not all applications are compatible with virtualization.
  1. High Initial Cost – Setting up a virtualized environment requires investment in powerful hardware.

Q6: Write steps to install and configure virtualization using KVM.

KVM (Kernel-Based Virtual Machine) is an open-source Linux hypervisor used to create and manage virtual machines.

Steps to Install and Configure KVM:

  1. Update System Packages:
    sudo apt update -y && sudo apt upgrade -y
  1. Install KVM and Required Packages:
    sudo apt install -y qemu-kvm libvirt-daemon libvirt-clients bridge-utils
  1. Start and Enable KVM Services:
    sudo systemctl enable --now libvirtd
  1. Verify KVM Installation:
    kvm-ok
  1. Create a Virtual Machine:
    virt-install --name=vm1 --ram=1024 --vcpus=1 --disk path=/var/lib/libvirt/images/vm1.img,size=10 --os-variant=ubuntu20.04 --cdrom=/path/to/ubuntu.iso
  1. Manage Virtual Machines Using virsh:
    virsh list --all
  1. Access VM Console:
    virsh console vm1

Q7: What is a hypervisor in Cloud Computing? Types of Hypervisors?

A hypervisor (Virtual Machine Monitor, VMM) is software that creates and manages virtual machines.

Types of Hypervisors:

  1. Type 1 Hypervisor (Bare Metal)
    • Runs directly on hardware.
    • Provides better performance and efficiency.
    • Examples: VMware ESXi, Microsoft Hyper-V, KVM, Xen.
  1. Type 2 Hypervisor (Hosted)
    • Runs on top of an existing OS.
    • Easier to set up but has performance overhead.
    • Examples: VMware Workstation, VirtualBox, Windows Virtual PC.
