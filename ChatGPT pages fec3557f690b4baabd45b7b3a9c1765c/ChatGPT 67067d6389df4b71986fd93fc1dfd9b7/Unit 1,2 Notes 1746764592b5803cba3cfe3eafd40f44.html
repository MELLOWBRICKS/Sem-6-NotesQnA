1) What is Activity Lifecycle and saving instance state? Explain in detail.

  • Activity Lifecycle: The Activity lifecycle in Android is a series of states an activity goes through from when it is created until it is destroyed. Understanding the lifecycle is crucial for effective memory management and UI behavior.
    • States:
      1. onCreate(): Called when the activity is first created. Used to initialize activity components.
      1. onStart(): Called when the activity becomes visible to the user.
      1. onResume(): Called when the activity starts interacting with the user.
      1. onPause(): Called when the system is about to start resuming another activity. Save data here.
      1. onStop(): Called when the activity is no longer visible to the user.
      1. onDestroy(): Called before the activity is destroyed.
  • Saving Instance State: Instance state is important for retaining data across configuration changes (like screen rotations):
    • Use the onSaveInstanceState(Bundle outState) method to save UI-related data. Example:
    @Override
protected void onSaveInstanceState(Bundle outState) {
    super.onSaveInstanceState(outState);
    outState.putString("MY_KEY", myData);
}
    • Restore it in onCreate(Bundle savedInstanceState).
    @Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    if (savedInstanceState != null) {
        myData = savedInstanceState.getString("MY_KEY");
    }
}

3) Explain what is implicit intent in detail.

  • Implicit Intent: Unlike explicit intents where the component is specified, implicit intents allow the Android system to find the appropriate component to handle the action.
    • Use cases:
      • Starting a web browser to view a webpage: Intent intent = new Intent(Intent.ACTION_VIEW, Uri.parse("<http://www.example.com>"));
      • Sending an email: Intent intent = new Intent(Intent.ACTION_SENDTO, Uri.parse("<mailto:recipient@example.com>"));
    • Activity Resolution: The system matches the intent with available components based on the action and data provided.
    • Advantages:
      • Promotes code reusability.
      • Allows for interaction between different applications.

4) Explain what is explicit intent in detail.

  • Explicit Intent: This type of intent specifies the exact component (i.e., activity) to start by providing the class name of the target activity.
    • Use cases:
      • Navigating within your own application:
      Intent intent = new Intent(this, SecondActivity.class);
startActivity(intent);
    • Component Naming: Can specify the package name and class name if the activity is in a different package.
    • Advantages:
      • Clear about which component is being called.
      • Suitable for tight coupling between components in the same app.

5) Explain what is TimePicker in detail.

  • TimePicker: A UI component for selecting a time (hour and minute). It can be presented either as a dialog or embedded in the layout.
    • Attributes:
      • android:timePickerMode: Choose between spinner and clock mode.
    • Usage Example:
    <TimePicker
    android:id="@+id/timePicker"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content" />
    • Getting values: Retrieve selected time from TimePicker.
    TimePicker timePicker = findViewById(R.id.timePicker);
int hour = timePicker.getHour();
int minute = timePicker.getMinute();

6) Explain what is DatePicker in detail.

  • DatePicker: A UI component that allows users to select a date. Like TimePicker, it can also appear as a dialog or embedded view.
    • Attributes:
      • android:datePickerMode: Choose between spinner and calendar mode.
    • Usage Example:
    <DatePicker
    android:id="@+id/datePicker"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content" />
    • Getting values: Retrieve selected date from DatePicker.
    DatePicker datePicker = findViewById(R.id.datePicker);
int day = datePicker.getDayOfMonth();
int month = datePicker.getMonth();
int year = datePicker.getYear();

7) Explain what is ListView with the help of 5 attributes and methods.

  • ListView: A view that displays a list of scrollable items and allows an adapter to manage the data.
    • Attributes:
      • android:divider: Customizes the divider between list items.
      • android:dividerHeight: Height of the divider.
      • android:layout_width: Width of the ListView.
      • android:layout_height: Height of the ListView.
      • android:choiceMode: Determines how multiple items can be selected.
    • Methods:
      • setAdapter(): Sets the adapter that provides the data.
      • setOnItemClickListener(): Responds when a list item is clicked.
      • getSelectedItem(): Returns the currently selected item.
      • getCount(): Returns the total number of items in the list.
      • setVisibility(): Controls the visibility of the ListView.

8) Explain what is Option menu with the help of methods, menu.xml file.

  • Option Menu: A primary menu for an activity that is displayed when the user presses the Menu button.
    • Creation: Defined in menu.xml within the res/menu folder.
    • Example menu.xml:
    <menu xmlns:android="<http://schemas.android.com/apk/res/android>">
    <item
        android:id="@+id/action_settings"
        android:title="Settings"
        android:orderInCategory="100"
        android:showAsAction="never" />
</menu>
    • Methods:
      • onCreateOptionsMenu(Menu menu): Inflate the menu resource.
      • onOptionsItemSelected(MenuItem item): Handle item selections.

9) Explain what is Context menu with the help of methods, menu.xml file.

  • Context Menu: A floating menu that appears when the user performs a long press on a view.
    • Creating: Registered with a view using registerForContextMenu(view) and created in menu.xml.
    • Example menu.xml:
    <menu xmlns:android="<http://schemas.android.com/apk/res/android>">
    <item
        android:id="@+id/action_delete"
        android:title="Delete" />
    <item
        android:id="@+id/action_edit"
        android:title="Edit" />
</menu>
    • Methods:
      • onCreateContextMenu(ContextMenu menu, View v, ContextMenu.ContextMenuInfo menuInfo): Inflate context menu.
      • onContextItemSelected(MenuItem item): Handle context menu item selections.

10) Explain what is PopUp menu with the help of methods, menu.xml file.

  • PopUp Menu: A type of context menu that shows a list of items when the user clicks an anchor view.
    • Creating: Can be initiated from any view and is useful to offer selections without obscuring the UI.
    • Example Popup menu creation:
    PopupMenu popup = new PopupMenu(this, anchorView);
popup.getMenuInflater().inflate(R.menu.menu_popup, popup.getMenu());
popup.setOnMenuItemClickListener(new PopupMenu.OnMenuItemClickListener() {
    @Override
    public boolean onMenuItemClick(MenuItem item) {
        return true;
    }
});
popup.show();
    • Menu methods: Similar to other menus, defined in menu.xml.

11) Explain what is Up and back screen navigation.

  • Up Navigation: Takes the user to the parent activity as defined in the Android manifest.
  • Back Navigation: Generally represented by the back button, allows users to return to the previous activity in the back stack.
  • Example of Up Navigation: To enable Up navigation, declare parent activities in the AndroidManifest.xml:
<activity
    android:name=".ChildActivity"
    android:label="@string/title_activity_child">
    <meta-data
        android:name="android.support.PARENT_ACTIVITY"
        android:value=".ParentActivity"/>
</activity>
  • Handling in code: Use NavUtils.navigateUpFromSameTask(this); to handle Up navigation.

12) Explain what is a service?

  • Service: A component that runs in the background to perform long-running operations or to perform work for remote processes. They don't have a user interface.
  • Types of services:
    • Started Service: A service that is started when an application component calls startService().
    • Bound Service: A service that allows components to bind to it using bindService(), facilitating communication.
  • Example Start a service:
Intent serviceIntent = new Intent(this, MyService.class);
startService(serviceIntent);
  • Defining a Service:
public class MyService extends Service {
    @Override
    public void onCreate() {
        super.onCreate();
    }
}

13) Explain what are Themes, Styles, Drawables, Material design in Android.

  • Themes: Define the overall look and feel of an application. Set in the AndroidManifest.xml or programmatically.
  • Styles: A collection of attributes applied to a single view or a group of views. Defined in res/values/styles.xml.
  • Drawables: Used for backgrounds, non-interactive content or UI elements, defined in drawable resource files (XML, PNG).
  • Material Design: A design language developed by Google that combines classic principles of good design with innovation in technology and science, emphasizing responsive animations, grid-based layouts, and depth effects.

14) Explain Connecting to Internet in Android in detail.

  • Connectivity Overview: Android provides several ways to connect to the internet based on user needs.
  • WiFi Connection: Make use of WifiManager. Example:
WifiManager wifiManager = (WifiManager) getApplicationContext().getSystemService(Context.WIFI_SERVICE);
wifiManager.setWifiEnabled(true);
  • Network Connection: Using ConnectivityManager, check the active network. Example:
ConnectivityManager cm = (ConnectivityManager) context.getSystemService(Context.CONNECTIVITY_SERVICE);
NetworkInfo activeNetwork = cm.getActiveNetworkInfo();
boolean isConnected = activeNetwork != null && activeNetwork.isConnectedOrConnecting();
  • Permissions: Necessary permissions should be defined in AndroidManifest.xml:
<uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE"/>

15) Explain what is a BroadcastReceiver in detail.

  • BroadcastReceiver: A component that listens for and responds to broadcast messages from other applications or from the system itself.
  • Use cases: To receive and handle intents such as device boot completion or network connectivity changes.
  • Example Declaration: Define in AndroidManifest.xml:
<receiver android:name=".MyBroadcastReceiver">
    <intent-filter>
        <action android:name="android.intent.action.BOOT_COMPLETED"/>
    </intent-filter>
</receiver>
  • Handling broadcasts: Create a subclass of BroadcastReceiver and override onReceive() method:
public class MyBroadcastReceiver extends BroadcastReceiver {
    @Override
    public void onReceive(Context context, Intent intent) {
        // Handle the received broadcast
    }
}
