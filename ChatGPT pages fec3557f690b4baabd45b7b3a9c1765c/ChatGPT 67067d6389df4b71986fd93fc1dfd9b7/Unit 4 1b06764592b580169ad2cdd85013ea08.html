Q1. With the help of a diagram, explain MAC Spoofing and Flooding in detail.

Definition:

"MAC Spoofing is a technique where an attacker alters the Media Access Control (MAC) address of their network device to impersonate another device on the network. MAC Flooding is an attack where an attacker sends numerous packets with different MAC addresses to a switch, exhausting its memory and forcing it into a fail-open mode, allowing packet sniffing."

MAC Spoofing

  • Attacker changes their MAC address to bypass network security policies.
  • Used for identity theft, evading access control, and Man-in-the-Middle (MITM) attacks.
  • Tools: MAC Changer, SMAC, Technitium MAC Address Changer.

MAC Flooding

  • Overloads a switchâ€™s MAC address table, converting it into a hub.
  • Attacker can capture unicast traffic not meant for them.
  • Used in network sniffing and MITM attacks.

Q5. Explain in brief Steganography with respect to hacking.

Definition:

"Steganography is the practice of concealing information within digital files, such as images, audio, or video, to avoid detection."

Steganography in Hacking:

  • Data Concealment: Attackers embed malicious payloads within digital media.
  • Evasion of Security Measures: Hidden data bypasses firewalls and antivirus systems.
  • Common Techniques:
    1. Least Significant Bit (LSB) Manipulation â€“ Alters pixel values in images to store data.
    1. Metadata Injection â€“ Stores hidden code in file properties (e.g., EXIF data in images).
    1. Whitespace Steganography â€“ Uses invisible characters in text files to hide information.

Example:

An attacker embeds a malicious script inside an image file and distributes it via email. When executed, the script extracts and runs the hidden payload.

Prevention Techniques:

  • Use steganalysis tools like StegExpose and Stegdetect.
  • Monitor unusual file sizes and metadata inconsistencies.
  • Implement strict file integrity checks and anomaly detection.

Q7. Write a short note on SYN Flooding.

Definition:

"SYN flooding is a Denial-of-Service (DoS) attack where an attacker sends numerous SYN requests to a server without completing the handshake, exhausting resources."

How It Works:

  1. Attacker sends multiple SYN requests.
  1. Server allocates resources for each request.
  1. Attacker never completes the handshake, causing resource exhaustion.

Prevention Methods:

  • SYN Cookies â€“ Validates requests before allocating resources.
  • Rate Limiting â€“ Limits SYN requests per second.
