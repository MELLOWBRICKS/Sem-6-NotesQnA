Q1. What are the Phases of Ethical Hacking? Explain in Detail.

Definition:

"Ethical hacking follows a structured approach to systematically identify vulnerabilities and strengthen security measures. The key phases include reconnaissance, scanning, gaining access, maintaining access, and covering tracks, each of which is crucial in simulating real-world attacks and fortifying defenses."

Phases of Ethical Hacking:

  1. Reconnaissance (Footprinting & Enumeration)
    • Collects preliminary information about the target.
    • Involves active (direct interaction) and passive (indirect observation) reconnaissance.
  1. Scanning
    • Identifies live hosts, open ports, and vulnerabilities.
    • Uses tools like Nmap, Nessus, and Wireshark.
  1. Gaining Access
    • Exploits vulnerabilities to breach the system.
    • Methods include phishing, password cracking, buffer overflow, and SQL injection.
  1. Maintaining Access
    • Ensures persistence inside the compromised system.
    • Uses techniques like backdoors, Trojans, and rootkits.
  1. Covering Tracks
    • Deletes logs and hides malicious activity.
    • Uses steganography and event log alteration.

Q2. Explain the Steps of Enterprise Strategy.

Definition:

"Enterprise security strategy involves repeated penetration testing, continuous security monitoring, and systematic approval by security teams to ensure robust application security."

Steps of Enterprise Strategy:

  1. Repeated Penetration Testing
    • Conducts periodic security assessments.
    • Identifies new vulnerabilities in evolving infrastructures.
  1. Approval by Security Testing Team
    • Ensures compliance with security policies.
    • Validates security measures before deployment.
  1. Continuous Application Security Testing
    • Implements real-time monitoring and automated security checks.
    • Uses tools like WebInspect, Nessus, and Qualys.

Q3. Explain the Term Security Testing and List Down Any 5 of Them with Its Description.

Definition:

"Security testing is a process to identify and mitigate security risks in software, ensuring that systems are protected against vulnerabilities, threats, and attacks."

Types of Security Testing:

  1. Penetration Testing
    • Simulates real attacks to find exploitable vulnerabilities.
  1. Vulnerability Assessment
    • Identifies and classifies security weaknesses in a system.
  1. Ethical Hacking
    • Simulates attacks in an authorized manner to strengthen security.
  1. Risk Assessment
    • Evaluates potential security risks and their impact.
  1. Security Auditing
    • Reviews security policies, controls, and compliance adherence.

Q4. Write a Short Note on Spidering with a Suitable Example.

Definition:

"Spidering is an automated process used by ethical hackers and web crawlers to navigate a website, following links to map out its structure and identify vulnerabilities."

Explanation:

  • Used in web penetration testing to find hidden directories.
  • Helps in discovering input fields vulnerable to injection attacks.
  • Tools like Burp Suite and OWASP ZAP perform spidering efficiently.

Example:

A hacker uses Burp Suite’s Spidering tool to map an e-commerce website and discovers an unprotected admin login panel.

Q6. Write a Short Note on Crawling with a Suitable Example.

Definition:

"Crawling is a technique used in web security testing where an automated bot or script systematically browses a website to discover pages, files, and vulnerabilities."

Explanation:

  • Used to map the entire website structure.
  • Helps in finding hidden pages and exposed directories.
  • Supports vulnerability detection by identifying attack vectors.

Example:

A security tester uses Google Dorking and a web crawler to find exposed confidential files on a government website.

Q8. Explain the Importance of Internal vs. External Penetration Testing in Ethical Hacking.

Definition:

"Internal penetration testing assesses security within an organization's network, while external penetration testing evaluates security from an outside attacker's perspective."

Importance of Internal Penetration Testing:

  1. Identifies Insider Threats – Detects security loopholes exploitable by employees.
  1. Tests Internal Network Security – Evaluates firewalls and access controls.
  1. Simulates Malicious Insiders – Assesses risks from compromised internal accounts.

Importance of External Penetration Testing:

  1. Detects External Threats – Evaluates exposure to internet-based attacks.
  1. Tests Perimeter Security – Assesses firewalls, VPNs, and external services.
  1. Simulates Real Cyber Attacks – Helps organizations strengthen security from an attacker’s perspective.
